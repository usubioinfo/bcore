what can we do

Due to a lack of well annotated references available for any given genome, using a De Novo Transcriptome Assembly is the way to go.

During the assembly, we expect to encounter the partition of many individual disconnected graphs that will need to be processed independently by the assembler in order to extract full-length isoforms that contribute to all plausible transcript sequences. In many cases we use assemblers such as SPAdes, Trinity, Abyss, and Sickle. We predict that for the project to be successful the assembler needs to be suitable for the given reads and also be able to do:

  • Raw sub-reads overlapping for error correction
  • Pre-assembly and error correction
  • Overlapping detection of the error corrected reads
  • Overlap filtering
  • Construct graph from overlaps
  • Construct contig from graph

" This method, termed RNA-Seq (RNA sequencing), has clear advantages over existing approaches and is expected to revolutionize the manner in which eukaryotic transcriptomes are analysed. "

Wang, Gerstein, and Snyder

Algorithm Overview:

Most assemblers have a set of scripts to execute all the discussed steps above. They usually require a large capacity memory(at least 32 core cpus and 128Gb RAM to assemble a 100M size genome with a reasonable amount of time). However, the assembler we choose should show efficiency for the type of reads sequences being analyzed. They also offer variety of other options either to do filtering or to control length-cutoffs, and the number of Mb sequence per block. Though the assembler of choice for any given project depends on your choosing, it is also useful to mention that not one assembler is the only tool capable of constructing de Bruijn graphs and report transcripts when it comes to De novo assembly. If needs to be, other assemblers should be tested and considered.
